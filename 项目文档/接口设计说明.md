# 接口设计说明（API Spec）：信息安全密码算法接口化实现与可测验证平台

> 技术栈：Python + Flask（RESTful JSON API）  
> 目标：满足期中作业要求——“第三方程序可通过接口调用并返回结果及执行状态代码”，并做到“输入输出及执行过程可见及可测”。  
> 约定：所有核心密码算法均为**从零实现**；接口只负责参数解析、校验、调度与统一输出。

---

## 1. API 总则

### 1.1 Base URL
- 本地开发：`http://127.0.0.1:5000`
- API 前缀：`/api`

### 1.2 Content-Type
- 请求：`Content-Type: application/json`
- 响应：`application/json; charset=utf-8`

### 1.3 通用请求字段（所有接口一致）
为方便第三方调用与复现实验，所有算法接口建议采用以下通用字段：

- `encoding`：输入数据编码（默认 `hex`）
  - `hex`：十六进制字符串（不含 `0x`，大小写均可）
  - `base64`：Base64 字符串
  - `text`：普通字符串（按 UTF-8 编码为 bytes）
- `trace`：是否返回执行过程（默认 `false`）
- `trace_level`：可选，默认 `1`
  - `1`：摘要级 trace（所有算法支持）
  - `2`：过程级 trace（仅 AES/SHA256/RSA/ECDSA 支持；其他算法若传 2，会降级为 1 并提示）
- `output`：输出编码（默认 `hex`，可选 `base64` / `text` 视算法而定）

> 注意：为了“输入输出可见”，服务端会在 trace 中回显输入摘要，但会进行截断或脱敏（例如 key 只显示前后若干字节）。

### 1.4 通用响应结构（所有接口一致）
```json
{
  "status_code": 0,
  "status_message": "OK",
  "result": { },
  "trace": { },
  "time_ms": 12.34,
  "request_id": "optional-uuid"
}
```

字段说明：
- `status_code`：执行状态码（详见《状态码设计.md》）
- `status_message`：状态说明
- `result`：算法结果（结构随接口变化）
- `trace`：当 `trace=true` 时返回，否则可省略或为 `null`
- `time_ms`：耗时（毫秒）
- `request_id`：可选，便于日志定位

---

## 2. 数据字段编码约定

### 2.1 bytes 输入字段常用命名
- `data`：待处理数据（明文/消息）
- `key`：密钥
- `iv` / `nonce`：初始化向量或随机数（按算法需要）
- `salt`：PBKDF2 盐

### 2.2 bytes 输出字段常用命名
- `ciphertext` / `plaintext`
- `digest`
- `signature`
- `derived_key`

---

## 3. 接口清单（按四大类）

> 路由命名原则：`/api/<category>/<algorithm>/<operation>`  
> 其中 `<category>` 为 `symmetric`/`hash`/`encoding`/`asymmetric`。

# A. 对称加密（AES / SM4 / RC6）

## A1. AES 加密
**POST** `/api/symmetric/aes/encrypt`

### 请求 JSON
```json
{
  "data": "00112233445566778899aabbccddeeff",
  "key":  "000102030405060708090a0b0c0d0e0f",
  "mode": "ECB",
  "padding": "pkcs7",
  "encoding": "hex",
  "output": "hex",
  "trace": true,
  "trace_level": 2
}
```

参数说明：
- `mode`：建议至少支持 `ECB`、`CBC`（若实现了 CTR/GCM 可扩展）
- `padding`：`none` / `pkcs7`（ECB/CBC 常用）
- `iv`：当 `mode=CBC` 时必填（16 bytes）

### 响应 JSON（示例）
```json
{
  "status_code": 0,
  "status_message": "OK",
  "result": {
    "ciphertext": "69c4e0d86a7b0430d8cdb78070b4c55a",
    "output_encoding": "hex"
  },
  "trace": {
    "algorithm": "AES-128",
    "mode": "ECB",
    "padding": "pkcs7",
    "input_summary": { "len": 16, "encoding": "hex", "head_hex": "0011223344556677" },
    "key_summary": { "len": 16, "head_hex": "0001020304050607" },
    "steps": [
      { "name": "key_expansion", "round_keys_head": ["..."] },
      { "name": "round_state_sample", "round": 0, "state_hex": "..." },
      { "name": "round_state_sample", "round": 1, "state_hex": "..." }
    ]
  },
  "time_ms": 3.21
}
```

---

## A2. AES 解密
**POST** `/api/symmetric/aes/decrypt`

请求字段同 encrypt，`data` 代表密文；返回 `plaintext`。

---

## A3. SM4/RC6 加解密
- **POST** `/api/symmetric/sm4/encrypt`
- **POST** `/api/symmetric/sm4/decrypt`
- **POST** `/api/symmetric/rc6/encrypt`
- **POST** `/api/symmetric/rc6/decrypt`

> SM4/RC6 默认只要求 Level 1 trace（摘要），但接口字段保持一致，便于统一调用。

# B. 哈希 / HMAC / PBKDF2

## B1. 计算哈希摘要（统一入口）
**POST** `/api/hash/digest`

### 请求 JSON
```json
{
  "algorithm": "SHA256",
  "data": "616263",
  "encoding": "hex",
  "output": "hex",
  "trace": true,
  "trace_level": 2
}
```

支持的 `algorithm`：
- `SHA1` / `SHA256` / `SHA3` / `RIPEMD160`

### 响应 JSON（示例）
```json
{
  "status_code": 0,
  "status_message": "OK",
  "result": {
    "digest": "ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad",
    "output_encoding": "hex"
  },
  "trace": {
    "algorithm": "SHA256",
    "input_summary": { "len": 3, "encoding": "hex", "head_hex": "616263" },
    "steps": [
      { "name": "padding", "padded_len": 64, "bit_len": 24 },
      { "name": "message_schedule_sample", "t": 0, "w_hex": "..." }
    ]
  },
  "time_ms": 1.07
}
```

> 若 `algorithm=SHA256` 且 `trace_level=2`，返回过程级 trace；其他哈希会降级为 Level 1（或你也可后续扩展）。
