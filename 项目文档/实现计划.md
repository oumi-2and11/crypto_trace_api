## 三、实现阶段（6 个里程碑）

### 里程碑 1：公共基础设施 + 统一响应框架

**目标**：搭建所有后续算法依赖的公共模块，打通 API 请求-响应链路。

**交付物**：

1. **`utils/common/encoding.py`** — 统一编解码工具
   - `hex_to_bytes(hex_str) → bytes`
   - `bytes_to_hex(data) → str`
   - `base64_to_bytes(b64_str) → bytes`
   - `bytes_to_base64(data) → str`
   - `text_to_bytes(text) → bytes`（UTF-8）
   - `bytes_to_text(data) → str`
   - `parse_input(raw, encoding) → bytes`：根据 `encoding` 字段自动路由
   - `format_output(data, encoding) → str`：根据 `output` 字段格式化

2. **`utils/common/validation.py`** — 参数校验与状态码映射
   - 定义 `CryptoError` 异常类（携带 `status_code` + `status_message`）
   - 校验函数：`validate_encoding()`, `validate_key_length()`, `validate_mode()` 等
   - 状态码常量：与 `状态码设计.md` 对齐

3. **`utils/common/trace.py`** — TraceCollector
   - `TraceCollector` 类：`add_step(name, data)`, `to_dict()`
   - Level 1 自动摘要：算法名、操作、输入摘要、参数、输出摘要、耗时
   - Level 2 手动记录：关键中间步骤（由算法实现主动调用）
   - 截断/脱敏：长数据只保留前后 N 字节，key 只显示前 8 字节

4. **`utils/common/timing.py`** — 耗时统计
   - `@measure_time` 装饰器或上下文管理器，返回 `time_ms`

5. **`utils/common/__init__.py`** — 导出公共接口

6. **统一 API 响应封装**（在 `webapp/` 层）
   - `make_response(status_code, status_message, result=None, trace=None, time_ms=None) → dict`
   - 所有 API 视图函数使用此封装

7. **一个端到端验证接口**（可选）
   - 用 Base64 encode/decode 作为第一个打通的 API，验证整条链路

**预计工作量**：~1 天

### 里程碑 2：编码算法 + SHA-256（含 Level 2 Trace）+ 自检框架

**目标**：实现最简单的算法（Base64/UTF-8）和最核心的哈希（SHA-256），搭建自检框架。

**交付物**：

1. **`utils/encoding/base64_codec.py`** — Base64 从零实现
   - `encode(data: bytes) → str`：标准 Base64 编码表
   - `decode(encoded: str) → bytes`：含 padding 校验与非法字符检测
   - 不使用 Python 内置 `base64` 模块

2. **`utils/encoding/utf8_codec.py`** — UTF-8 编解码
   - `encode(text: str) → bytes`：字符 → UTF-8 字节序列
   - `decode(data: bytes) → str`：字节序列 → 字符，含非法序列错误处理
   - 不使用 `text.encode('utf-8')`，自行实现编码规则（1-4 字节）

3. **`utils/hash/sha256.py`** — SHA-256 从零实现
   - `digest(data: bytes, trace_collector=None) → bytes`
   - 常量初始化（K[0..63], H[0..7]）
   - 消息填充（padding）：按 FIPS 180-4 规范
   - 消息调度（W[0..63]）
   - 压缩函数：64 轮运算
   - **Level 2 Trace**：记录 padding 详情、W[t] 摘要、若干轮 a..h 寄存器采样

4. **`utils/hash/__init__.py`** — 重构为 per-algorithm 结构
   - 拆分 `basic_hash.py` → `sha1.py`, `sha256.py`, `sha3.py`, `ripemd160.py`
   - 暂时保留占位桩，后续里程碑逐个实现

5. **`utils/tests/vectors.py`** — 测试向量管理
   - 数据结构：`{ algorithm: { "input": ..., "params": ..., "expected": ... } }`
   - 先填入 Base64、UTF-8、SHA-256 的标准测试向量
   - 后续每个里程碑追加对应算法的向量

6. **`utils/tests/selftest.py`** — 自检引擎
   - `run_selftest(algorithms=None) → dict`：执行指定算法的测试向量，返回 PASS/FAIL 汇总
   - 输出结构：`{ algorithm: { "status": "PASS"/"FAIL", "actual": ..., "expected": ..., "time_ms": ... } }`

7. **API 端点**（新增）：
   - `POST /api/encoding/base64/encode`
   - `POST /api/encoding/base64/decode`
   - `POST /api/encoding/utf8/encode`
   - `POST /api/encoding/utf8/decode`
   - `POST /api/hash/digest`（仅 SHA-256 可用，其余返回"待实现"）
   - `POST /api/selftest/run`

8. **前端页面交互改造**（编码页面 + SHA-256 页面）
   - 表单：输入框 + 参数选择 + 执行按钮
   - 结果展示：输出 + Trace 展示区
   - AJAX 调用对应 API

**预计工作量**：~2-3 天

### 里程碑 3：AES（含 Level 2 Trace）+ 其余哈希/HMAC/PBKDF2（Level 1）

**目标**：完成对称加密核心算法，补齐哈希类算法。

**交付物**：

1. **`utils/symmetric/aes.py`** — AES-128 从零实现
   - `encrypt(plaintext, key, mode='ECB', iv=None, padding='PKCS7', trace_collector=None) → bytes`
   - `decrypt(ciphertext, key, mode='ECB', iv=None, padding='PKCS7', trace_collector=None) → bytes`
   - S-Box / InvS-Box 查表
   - KeyExpansion：轮密钥扩展
   - 轮函数：SubBytes → ShiftRows → MixColumns → AddRoundKey
   - 逆轮函数：InvShiftRows → InvSubBytes → AddRoundKey → InvMixColumns
   - 工作模式：ECB / CBC（可选 CTR）
   - PKCS7 填充
   - **Level 2 Trace**：轮密钥摘要、每轮 state 采样（可配置采样轮数）

2. **`utils/hash/sha1.py`** — SHA-1 从零实现（Level 1 Trace）
   - 结构与 SHA-256 类似，80 轮，5 个寄存器

3. **`utils/hash/sha3.py`** — SHA-3 (Keccak) 从零实现（Level 1 Trace）
   - Keccak-f[1600] 置换函数
   - 吸收-挤压 (sponge) 模式
   - SHA3-256 输出

4. **`utils/hash/ripemd160.py`** — RIPEMD-160 从零实现（Level 1 Trace）
   - 双路（左/右）5 轮 × 16 步

5. **`utils/mac_kdf/`** — 新建子包
   - `hmac.py`：`compute(key, data, hash_algo='sha256') → bytes`（Level 1 Trace）
   - `pbkdf2.py`：`derive(password, salt, iterations, dk_len, hash_algo='sha256') → bytes`（Level 1 Trace）
   - 内部调用自写的 SHA-1/SHA-256

6. **API 端点**（新增）：
   - `POST /api/symmetric/aes/encrypt`
   - `POST /api/symmetric/aes/decrypt`
   - `POST /api/hash/digest`（扩展支持 SHA-1/SHA-3/RIPEMD-160）
   - `POST /api/hmac/compute`
   - `POST /api/kdf/pbkdf2`

7. **测试向量追加**：AES、SHA-1、SHA-3、RIPEMD-160、HMAC、PBKDF2

8. **前端页面改造**：AES 页面 + 各哈希/HMAC/PBKDF2 页面

**预计工作量**：~3-4 天

### 里程碑 4：RSA + ECC + ECDSA（含 Level 2 Trace）

**目标**：完成公钥密码全部算法，这是实现难度最高的阶段。

**交付物**：

1. **`utils/asymmetric/rsa.py`** — RSA-1024 从零实现
   - `keygen(bits=1024) → (public_key, private_key)`
     - 素数生成：Miller-Rabin 素性测试
     - 大整数运算：`gcd`, `modinv`（扩展欧几里得）
   - `encrypt(plaintext, public_key) → bytes`（可选，按作业要求）
   - `decrypt(ciphertext, private_key) → bytes`（可选）
   - `sign(message, private_key, hash_algo='sha256') → bytes`
   - `verify(message, signature, public_key, hash_algo='sha256') → bool`
   - **Level 2 Trace**：p/q 位数、n 位数、e/d 生成摘要、签名/验签关键步骤

2. **`utils/asymmetric/rsa_sha1.py`** — RSA-SHA1
   - `sign(message, private_key) → bytes`：用自写 SHA-1 + RSA 签名
   - `verify(message, signature, public_key) → bool`
   - **Level 2 Trace**：SHA-1 摘要 + PKCS#1 v1.5 padding + 模幂运算摘要

3. **`utils/asymmetric/ecc.py`** — ECC-160 从零实现
   - 有限域运算：`Fp` 加/减/乘/逆
   - 椭圆曲线点运算：点加、倍点、标量乘
   - 曲线选择：secp160r1（或自定义 160-bit 曲线，报告中说明）
   - `keygen() → (public_key_point, private_key_int)`
   - **Level 2 Trace**：曲线参数、点运算中间量

4. **`utils/asymmetric/ecdsa.py`** — ECDSA 签名
   - `sign(message_hash, private_key) → (r, s)`
   - `verify(message_hash, signature, public_key) → bool`
   - 内部调用 ECC 模块
   - **Level 2 Trace**：k 生成摘要、点运算摘要、(r,s) 值、验签判断过程

5. **API 端点**（新增）：
   - `POST /api/asymmetric/rsa/keygen`
   - `POST /api/asymmetric/rsa-sha1/sign`
   - `POST /api/asymmetric/rsa-sha1/verify`
   - `POST /api/asymmetric/ecc/keygen`
   - `POST /api/asymmetric/ecdsa/sign`
   - `POST /api/asymmetric/ecdsa/verify`

6. **测试向量追加**：RSA、ECC、ECDSA（公钥算法向量可用固定种子生成确定性结果）

7. **前端页面改造**：RSA、RSA-SHA1、ECC、ECDSA 页面

**预计工作量**：~3-4 天（大整数与有限域运算实现较复杂）

### 里程碑 5：SM4 + RC6（Level 1）

**目标**：补齐剩余对称加密算法。

**交付物**：

1. **`utils/symmetric/sm4.py`** — SM4 从零实现（Level 1 Trace）
   - `encrypt(plaintext, key, trace_collector=None) → bytes`
   - `decrypt(ciphertext, key, trace_collector=None) → bytes`
   - S-Box、系统参数 FK、固定参数 CK
   - 32 轮轮函数 F
   - 密钥扩展：32 轮轮密钥
   - 支持 ECB/CBC 模式

2. **`utils/symmetric/rc6.py`** — RC6 从零实现（Level 1 Trace）
   - `encrypt(plaintext, key, trace_collector=None) → bytes`
   - `decrypt(ciphertext, key, trace_collector=None) → bytes`
   - RC6-32/20/b（w=32, r=20）
   - 密钥扩展：从 b 字节密钥生成 2r+4 个轮密钥
   - 轮函数：加法、异或、乘法、旋转
   - 支持 ECB/CBC 模式

3. **API 端点**（新增）：
   - `POST /api/symmetric/sm4/encrypt`
   - `POST /api/symmetric/sm4/decrypt`
   - `POST /api/symmetric/rc6/encrypt`
   - `POST /api/symmetric/rc6/decrypt`

4. **测试向量追加**：SM4（国密标准向量）、RC6

5. **前端页面改造**：SM4、RC6 页面

**预计工作量**：~2 天

## 五、风险与注意事项

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| RSA/ECC 大整数运算复杂 | 里程碑 4 可能延期 | 先用 Python 原生大整数（`int`），避免手动实现大数库；Miller-Rabin 用确定性基 |
| SHA-3 (Keccak) 状态机实现复杂 | 里程碑 3 阻塞 | 可延后至里程碑 5，先完成 HMAC/PBKDF2 |
| SM4/RC6 测试向量稀缺 | 里程碑 5 验证困难 | SM4 用国密标准附录向量；RC6 用论文原始向量 |
| 前端交互工作量被低估 | 全部页面需改造 | 优先做算法页面，about/selftest/error 保持静态；使用统一 JS 模板 |
| utils/ 目录结构重构导致导入断链 | 影响已有代码 | 分步重构，每步确保 `python -c "from utils.xxx import yyy"` 可运行 |
